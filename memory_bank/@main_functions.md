# LLM News Tracker — 功能规格书

## 1. 产品概述

- **一句话定位**：自动监控 YouTube LLM 领域热门频道的智能聚合器，用 AI 对视频内容做真实摘要和主题分类，以在线表格形式公开呈现。
- **目标用户**：AI 从业者、研究人员、技术管理者，想要高效追踪 LLM 前沿动态但不想手动刷多个频道的人。
- **核心价值主张**：不看标题看内容——每段视频都由 AI 分析字幕后生成真实摘要，而非依赖标题/封面；频道自动发现机制确保覆盖「最活跃 + 最受欢迎」的 LLM 内容源。

## 2. 功能清单

### P0 — MVP 必需

| 功能模块 | 描述 | 验收标准 |
|---------|------|---------|
| 频道管理 | 维护种子频道列表（5-10 个初始 LLM 频道），支持从 YouTube 相关推荐自动发现新频道 | 种子频道可配置；每次运行自动评估关联频道是否值得加入 |
| 视频抓取 | 通过 RSS Feed 获取频道最新视频列表，下载视频字幕（含 YouTube 自动生成字幕） | 每次运行能获取所有监控频道最近 30 天的视频 |
| AI 内容分析 | 调用 Gemini API 对字幕文本做：主题分类（微调/RAG/Agent/行业应用等）、内容摘要、主讲人识别 | 每个视频产出结构化分析结果：主题标签 + 150 字以内摘要 + 主讲人 |
| 数据表格页面 | 公开网页以表格展示所有视频，包含：频道/作者、标题、发布时间、播放量、主题标签、AI 摘要、频道间关联分析 | 页面可通过 GitHub Pages URL 公开访问 |
| 搜索/筛选 | 表格支持按关键词搜索、按主题标签筛选、按频道筛选 | 搜索结果实时过滤，响应时间 < 200ms |
| 定时自动更新 | GitHub Actions 每 12 小时触发一次全流程：抓取 → 分析 → 更新页面 → 部署 | 页面内容与 YouTube 实际更新延迟不超过 12 小时 |
| 频道关联分析 | 分析不同频道在 LLM 话题上的内容侧重、观点互补或对立关系，在表格区域展示 | 每个频道有一份简短的内容侧重描述 + 与其他频道的关联说明 |

### P1 — 后续迭代

| 功能模块 | 描述 | 优先级 |

## 3. 用户故事

### 故事 1：AI 工程师想要快速了解本周 LLM 动态
**Given** 我是 AI 工程师，没有时间每天刷 10 个 YouTube 频道
**When** 我打开 LLM News Tracker 页面
**Then** 我看到一张表格，按发布时间倒序展示所有监控频道的视频，每条都有 AI 自动生成的摘要，我可以按「RAG」「Agent」等主题快速筛选

### 故事 2：技术 Leader 想要对比不同频道的观点
**Given** 我关注多个 LLM 频道，想知道它们分别侧重什么
**When** 我查看页面上的频道关联分析区域
**Then** 我看到每个频道的内容侧重描述，以及它和其他频道在话题上的互补或对立关系

### 故事 3：系统自动发现新频道
**Given** LLM 领域不断有新创作者涌现
**When** 系统每 12 小时运行一次更新
**Then** 系统自动评估关联频道是否活跃（7 天内有新视频）且播放量高，符合条件的自动纳入监控

## 4. 数据模型概览

### 核心实体

```
Channel（频道）
├── channel_id: 唯一标识
├── name: 频道名称
├── rss_url: RSS Feed 地址
├── content_focus: AI 分析的内容侧重描述
├── is_seed: 是否种子频道
├── active_score: 活跃度评分
└── added_at: 加入监控的时间

Video（视频）
├── video_id: 唯一标识
├── channel_id → Channel
├── title: 原始标题
├── url: 视频链接
├── published_at: 发布时间
├── view_count: 播放量（抓取时快照）
├── duration: 时长
├── has_captions: 是否有字幕
└── fetched_at: 抓取时间

Analysis（分析结果）—— 一个 Video 对应一条
├── video_id → Video
├── raw_transcript: 原始字幕文本（截断至 Gemini 上下文窗口）
├── summary: 150 字以内内容摘要
├── topics: 主题标签列表（如 ["RAG", "Fine-tuning"]）
├── speaker: 主讲人姓名
├── technical_level: 技术深度（入门/中级/高级）
├── analyzed_at: 分析时间

ChannelRelation（频道关联）
├── channel_a_id → Channel
├── channel_b_id → Channel
├── relation_type: 互补 / 对立 / 引用
├── description: 关联说明
├── generated_at: 生成时间
├── generated_by: 分析批次ID
```

### 关系概览
- Channel 1:N Video
- Video 1:1 Analysis
- Channel M:N ChannelRelation

## 5. 非功能性需求

### 性能要求
- 页面首次加载（含表格渲染）< 2 秒
- 搜索/筛选响应 < 200ms
- 单次全流程运行（抓取+分析+部署）< 10 分钟（GitHub Actions 限制）

### 安全考虑
- Gemini API Key 存储在 GitHub Secrets 中，不硬编码
- 页面不包含任何可执行后端代码，纯静态部署
- RSS Feed 读取为只读操作，不涉及认证

### 兼容性要求
- 支持 Chrome / Firefox / Safari / Edge 最新两个大版本
- 移动端响应式布局
- 表格在移动端可横向滚动

### 数据保留
- 视频记录保留最近 90 天
- 频道关联分析每次运行全量刷新
- 历史分析结果无需保留（每次可用最新字幕重新分析）
